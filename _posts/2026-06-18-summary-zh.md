## [OpenJDK 与 GraalVM 对 AI 代码政策截然相反](https://www.infoq.cn/article/hfTxUTc6pj2cHm1Hk5DD?utm_source=rss&utm_medium=article) ⭐️ 6.0/10

Oracle 旗下的 OpenJDK 和 GraalVM 项目对 AI 生成代码的贡献采取了截然相反的政策：OpenJDK 在一定条件下欢迎此类贡献，而 GraalVM 则明确禁止。 这种分歧凸显了开源社区关于 AI 在软件开发中作用的持续争论，可能影响其他项目制定自己的 AI 贡献政策。 OpenJDK 允许 AI 生成的代码，前提是贡献者证明其符合许可和原创性要求；而 GraalVM 则完全禁止任何 AI 生成的代码贡献，理由是担心代码质量和可维护性。

rss · InfoQ 中文 · 6月17日 14:30

**背景**: OpenJDK 是 Java 平台标准版的官方参考实现，而 GraalVM 是 Oracle Labs 开发的高性能 JDK 发行版，包含提前编译等高级特性。这两个项目都归 Oracle 所有，但治理和贡献指南不同。随着 GitHub Copilot 等工具的普及，关于 AI 生成代码在开源中的争论愈演愈烈，Zig 等项目也禁止了 AI 贡献。
